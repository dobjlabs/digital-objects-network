## Join the network

One line installs the driver daemon (`dobjd`), the `dobj` CLI, and the
[craft-basics](/applications/craft-basics) starter application into `~/.dobj/`:

```bash
curl -fsSL https://raw.githubusercontent.com/dobjlabs/zk-craft-releases/main/install.sh | sh
```

Windows, agent-driven setup, and manual instructions: **[Install the driver →](/install)**

## Public infrastructure

Fresh installs come pre-pointed at these hosted services, so the network works
out of the box:

| Service | Endpoint | Role |
| ------------ | ------------------------------------ | ---- |
| Synchronizer | [`sync.don.pateldhvani.com`](https://sync.don.pateldhvani.com/healthz) | Rebuilds canonical network state from Ethereum blobs; answers live / consumed queries |
| Relayer | [`relay.don.pateldhvani.com`](https://relay.don.pateldhvani.com/healthz) | Submits proof payloads to Ethereum as EIP-4844 blob transactions |

Your driver is free to point elsewhere — edit `~/.dobj/settings.json` and
restart the daemon:

```json
{
  "synchronizerApiUrl": "https://sync.don.pateldhvani.com",
  "relayerApiUrl": "https://relay.don.pateldhvani.com"
}
```

### Run your own — and list it here

Synchronizers and relayers are permissionless: anyone can run one against
their own Ethereum endpoints, any number can operate concurrently, and running
one confers no special network privileges — it just makes the network more
available. See the [architecture page](/architecture#network-architecture) for
what each service does.

## Applications

Objects and the proof-backed actions that govern them ship as plugins
(`.pexe` archives). Two examples are available today:

- ⛏️ **[craft-basics](/applications/craft-basics)** — a survival-crafting
  starter: find logs, refine wood, craft picks, mine stone. Bundled with the
  installer.
- 🚀 **[craft-rocket](/applications/craft-rocket)** — a 53-class factory tech
  tree across five tiers, culminating in a provable rocket. Built from source.
